Project 1 : Analog Sound Level Meter

Design and implementation of an analog sound level meter based on an electret microphone using only discrete electronic components (transistors, resistors, capacitors, LEDs and microphone). The system detects and visualizes sound intensity through a multi-threshold LED interface powered by a 9V supply.

Project Overview

This project focused on the development of a fully analog sound level meter capable of detecting sound levels between approximately 70 dB and 90 dB. The system processes extremely low-amplitude microphone signals and activates LEDs according to predefined voltage thresholds.

Analog Electronics Transistor-based Amplifiers LTSpice simulation PCB Design Signal Processing
View GitHub Repository →

Objective

Develop an analog sound measurement system using only discrete components while ensuring reliable threshold detection from millivolt-level microphone signals.

Methodology

Functional analysis, analog circuit design, LTSpice simulation, hardware prototyping, experimental validation and PCB manufacturing.

Responsibilities

Design of the signal processing chain, amplifier dimensioning, comparator implementation, simulation, debugging and PCB assembly.

System Architecture

The system architecture is composed of several functional blocks:

  • Electret microphone for signal acquisition
  • High-gain amplification stages
  • Voltage follower for impedance matching
  • Comparator stage with threshold detection
  • LED visualization system

LTSpice Circuit Design

Complete simulation and validation of the analog processing chain using LTSpice. The schematic includes amplification stages, impedance adaptation and transistor-based comparators.

Signal Processing & Amplification

  • Multi-stage transistor-based amplifier architecture.
  • Initial theoretical gain between 1500 and 2000, experimentally adjusted to approximately 400.
  • Measured behavior: 9 mVpp input → 3.75 Vpp output.
  • Signal distortion and negative clipping observed, but acceptable for peak-level detection.
  • Frequency cutoff measured around 50 Hz.

Threshold Detection

Three transistor-based comparators were used to detect different sound intensity levels. Voltage thresholds were generated using a resistor divider, enabling progressive LED activation depending on signal amplitude.

A Darlington configuration ensured a stable LED current close to 15 mA for reliable visualization.

Circuit Testing & Validation

All functional blocks were assembled and tested independently using an oscilloscope and a current probe.

PCB Design & Assembly

PCB Routing

PCB designed using Proteus with compact routing constraints (6.7 × 6.7 cm), mixed-layer routing and via integration.

Hardware Assembly

Manual assembly and soldering of all components onto the manufactured PCB followed by experimental validation and debugging.

Results & Analysis

Simulation Results

Successful detection of three sound levels, stable LED activation and identification of transient response limitations.

Experimental Results

Functional prototype with progressive LED activation and experimentally calibrated thresholds. Measured gain remained consistent with adjusted design.

Critical Issues

Final PCB validation was limited by false contacts caused by an inverted PCB layer during manufacturing, highlighting fabrication constraints.

Challenges & Engineering Decisions

  • Real vs theoretical component behavior
    Gain reduction and removal of one amplification stage after experimental validation.
  • Microphone sensitivity
    Empirical recalibration of threshold values.
  • Signal distortion & clipping
    Accepted because the application focused on peak detection rather than waveform fidelity.
  • PCB manufacturing defects
    Identification of practical fabrication constraints related to vias and layer orientation.

Project 2 : Ultrasonic Rotating Scanner

Embedded electronics and mechatronics project designed to build a 2D ultrasonic rotating scanner capable of measuring distances, reconstructing the shape of a room from angular scans, and exporting data for MATLAB visualization. The system combines embedded programming, electronic system design, mechanical integration and data processing.

Project Overview

The objective was to design and implement a rotating ultrasonic scanning system able to sweep an ultrasonic sensor over a controlled angular range, measure distances in real time, display the current state and measurement on an LCD screen, and export the acquired data for post-processing in MATLAB.

Embedded C++ Mechatronics Finite-State Machine MATLAB
The control logic is based on a robust finite-state machine architecture to manage start/stop actions, measurement cycles, and display updates with reliable system behavior.
View GitHub Repository →

Objective

Rotate an ultrasonic sensor over a controlled angular range, measure distances in real time and reconstruct the geometry of the scanned environment.

Hardware & Software

Stepper motor, ULN2003 driver, HC-SR04 sensor, LCD1602, 74HC595N shift register, push buttons, Arduino IDE, KiCad and MATLAB.

Key Challenge

Build a reliable embedded system while keeping the mechanical structure stable, reducing GPIO usage, and ensuring clean state transitions.

System Architecture

The complete architecture combines sensing, actuation, display and control. The stepper motor is driven by the Arduino through a ULN2003 driver module, while the LCD1602 display is controlled through a 74HC595N shift register in order to reduce the number of required GPIO pins.

  • Stepper motor for angular scanning
  • ULN2003 driver module for motor control
  • HC-SR04 ultrasonic sensor for distance measurement
  • LCD1602 display for state and data visualization
  • 74HC595N shift register for LCD communication
  • Two push buttons for Start / Stop control

Modeling / Theory

Circuit Scheme

The LCD is driven through a serial-to-parallel shift register to reduce Arduino pin usage, while the stepper motor is controlled through the ULN2003 interface.

Finite-State Machine

The control logic follows four operating states with Start and Stop inputs. This structure improves robustness and simplifies system supervision.

The scan cycle follows the sequence 0° → 180° → 0° → -180° → 0° ... in order to prevent cable twisting during continuous operation. The scan is performed with angular increments of 10°.

Angular Scanning FSM Control LCD Communication Cable Management

Implementation

The ultrasonic sensor is mounted on a vertically rotating pivot driven by the stepper motor. During scanning, the motor moves to each target angle, the HC-SR04 measures the distance, and the LCD displays the current state, angle and measured distance.

Motion Sequence

Motor rotation is performed in controlled angular steps to ensure repeatable measurements and coherent scan sampling.

Display Logic

The LCD displays the current state, angle and distance in real time, helping debug the embedded control loop.

System Robustness

Start/Stop push buttons and finite-state control improve operational stability and prevent unintended state changes.

Data Acquisition & Processing

A MATLAB script, Scanner2D.m, retrieves serial data from the Arduino and stores the measurements in a CSV file. A second script processes the CSV data and reconstructs the scanned object shape.

Experimental Results

Room Scan Reconstruction

The plot below shows the measured values obtained during the scan of a living room. False measurements larger than 500 cm were filtered out before processing. The reconstructed profile highlights the ability of the scanner to estimate the geometry of the surrounding environment from angular distance measurements.

The large beamwidth of the HC-SR04 ultrasonic sensor (approximately 15°) can clearly be observed, as nearby objects are detected over a relatively wide angular range. This behavior introduces measurement spreading and reduces spatial precision.

Validated System Functions

The scanner successfully demonstrated:

  • Accurate angular motion control
  • Real-time distance measurements
  • Serial export of scan data for post-processing
  • 2D room profile reconstruction from measured values

The project also demonstrated the integration of several engineering domains:

Embedded Systems Sensor Interfacing Motor Control Data Visualization
Although the prototype successfully validates the overall scanning architecture, the measurement performance remains limited by the relatively low precision and accuracy of the HC-SR04 ultrasonic sensor.

Challenges Encountered

Mechanical Integration

Designing a stable metallic structure was challenging because the motor and sensor were not originally intended for this type of assembly.

State Management

Button debouncing and the prevention of unintended multiple state transitions required careful software design.

Timing Constraints

The 74HC595N shift register required precise timing management to ensure stable LCD communication.

Sensor Limitations

The HC-SR04 sensor introduced noisy measurements, false detections and limited precision.
